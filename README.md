# Микросервис VehicleDocuments

Микросервис управления документами транспортных средств для проекта Wheel

## Описание

Микросервис VehicleDocuments предоставляет полный набор функций для управления документами транспортных средств в ИС для каршеринга. Сервис построен на основе Clean Architecture с использованием Domain-Driven Design (DDD) принципов.

## Архитектура

Проект следует принципам Clean Architecture и состоит из следующих слоев:

- **Api** - gRPC API слой с контрактами и мапперами
- **Application** - use cases, порты и обработчики доменных событий
- **Domain** - доменная логика, агрегаты и доменные события
- **Infrastructure** - реализация репозиториев и внешних сервисов

## Основные функции

### 📄 Управление документами транспортных средств
- **Создание сущности документов** - создание комплекта документов для транспортного средства
- **Добавление ПТС** - загрузка и управление паспортом транспортного средства
- **Добавление СТС** - загрузка и управление свидетельством о регистрации
- **Добавление ОСАГО** - загрузка и управление полисом обязательного страхования

### 🔍 Получение информации
- **Получение списка документов по ID транспортного средства** - получение полной информации о документах
- **Получение ссылки на S3 ПТС** - получение данных паспорта транспортного средства
- **Получение ссылки на S3 СТС** - получение данных свидетельства о регистрации
- **Получение ссылки на S3 ОСАГО** - получение данных полиса ОСАГО с информацией о сроке действия

## Доменные агрегаты

### VehicleDocuments
Основной агрегат, представляющий комплект документов транспортного средства:
- **Id** - уникальный идентификатор
- **SagaId** - идентификатор саги для координации процессов (инициатор саги — микросервис VehicleFleet при добавлении авто)
- **VehicleId** - идентификатор транспортного средства
- **Status** - статус комплектации документов
- **Pts** - паспорт транспортного средства
- **Sts** - свидетельство о регистрации

### Osago
Агрегат, представляющий полис обязательного страхования:
- **Id** - уникальный идентификатор
- **VehicleDocumentsId** - связь с документами транспортного средства
- **PhotoStorageBucketAndKey** - путь к фотографии документа в хранилище
- **ExpiryStatus** - статус истечения срока действия
- **DateOfIssue** - дата выдачи
- **DateOfExpiry** - дата истечения срока действия

## Статусы документов

### Статус комплектации документов
- **Комлексный статус**:
  - IsPtsAdded - ПТС добавлен
  - IsStsAdded - СТС добавлено
  - IsOsagoAdded - ОСАГО добавлено
- **AddingCompleted** - все документы добавлены

### Статус истечения ОСАГО
- **NotExpired** - не истек
- **Expired** - истек

## Технологический стек

- **.NET 8** - основная платформа
- **gRPC** - API протокол
- **PostgreSQL** - основная база данных
- **Kafka** - обмен сообщениями
- **MediatR** - паттерн Mediator для use cases
- **FluentResults** - обработка результатов операций
- **Dapper** - микро-ORM для запросов

## API Endpoints

Сервис предоставляет следующие gRPC методы:

### Команды
- `AddVehicleDocuments` - создание комплекта документов для транспортного средства
- `AddPtsToVehicleDocuments` - добавление паспорта транспортного средства
- `AddStsToVehicleDocuments` - добавление свидетельства о регистрации
- `AddOsago` - добавление полиса ОСАГО

### Запросы
- `GetVehicleDocumentsByVehicleId` - получение документов по ID транспортного средства
- `GetStsByVehicleDocumentsId` - получение СТС по ID документов
- `GetPtsByVehicleDocumentsId` - получение ПТС по ID документов
- `GetOsagoByVehicleDocumentsId` - получение ОСАГО по ID документов

## Доменные события

Система генерирует следующие доменные события:
- `DocumentAddingCompletedDomainEvent` - при завершении добавления всех документов (часть саги добавления авто VehicleFleet)
- `OsagoAddedDomainEvent` - при добавлении полиса ОСАГО
